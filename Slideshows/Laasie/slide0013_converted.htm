Our Observation:
-Application persistence requires prior knowledge of each specific application.
-
-Store changes to application state.
-
-How can we represent changes?
-
We can use a common infrastructure so that the developers can focus on application development. What we have observed so far is that 1. we need some information about the application schema. And 2. we should store the actual changes to an application’s state. So the question is what are those changes and how we can represent them .
