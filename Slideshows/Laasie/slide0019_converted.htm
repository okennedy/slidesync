Out of order Undo’s
-Laasie stores functional computations.
-
-Allows tracking of changes with time.
-
-The intermediate stages of data generation can be modified without affecting other stages.
This can be applied in the case where valid some changes are needed on the data which affects the changes made so far. If we save the raw data then we would have to figure out what those changes are and how they affect the current state. As we are now saving the changes itself out of order undo’s are generally simple to implement as we can get to the current state by re-computing all the functions.
