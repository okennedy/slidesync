Why Log Based Infrastructure?
-Natural representation of the State.
-Security, performance, and common functionalities for all applications developed on top.
-
-Server can choose between sending bulk updates or smaller transformations.
-
So the next question is why a log based infrastructure. By using logs we can provide a natural representation for any type of changes to the application’s state. As all the computations are saved in the form of individual changes we can delegate the computing to the various intermediate levels in a client server architecture.
